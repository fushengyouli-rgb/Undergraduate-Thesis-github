三、数据处理管道
================================================================================

【数据来源】
- 数据集: PhysioNet CTU-CHB产时胎心监护数据库
- 样本数: 552条记录（胎儿），75,952个切窗样本
- 数据划分: 训练集70%(386胎儿) / 验证集10%(55胎儿) / 测试集20%(111胎儿)
- 类别分布: 正常69,248样本，异常6,704样本（约8.8%）

【切窗参数】
- 窗口大小: 256点（约64秒，采样率4Hz）
- 步长: 128点（50%重叠）
- 双通道: FHR（胎心率50-200bpm）+ UC（宫缩0-100）

【预处理流程】
1. 异常值裁剪 - 限制在生理范围内
2. 中值滤波 - 核大小5（约1.25秒），去除beat halving/doubling伪影
3. 移动平均平滑 - 窗口7（约1.75秒）
4. 三次样条插值 - 补全缺失区域

四、轻量化模型架构
================================================================================

【模型规格】
- 参数量: ~33,105（远低于10万目标）
- 推理时间: ~37.20ms
- 输入维度: (256, 1)双通道 + 5维手工特征

【网络结构】
输入层
├─ FHR分支: Input(shape=(256,1))
├─ UC分支: Input(shape=(256,1))
└─ 手工特征: Input(shape=(5,))

FHR特征提取:
├─ Conv1D(16,1) + BatchNorm
├─ SepConv(16,5) + SE + MaxPool
├─ SepConv(32,5) + SE + MaxPool
├─ SepConv(48,3,dilation=2) × 2 + ChannelAttention
└─ GlobalAvgPool

UC特征提取:
├─ Conv1D(8,7) + BN + MaxPool(4)
└─ Conv1D(16,5) + GlobalAvgPool

时序建模:
├─ GRU(48, return_sequences=True)
└─ GRU(32)

输出: Concatenate → Dropout(0.6) → Dense(1, sigmoid)

【轻量化技术】
- 深度可分离卷积(SeparableConv1D): 减少参数量和计算量
- SE注意力机制: 增强特征筛选能力
- 通道注意力机制: 自适应强化异常信号区域
- L1正则化: weight_decay=5e-5
- 简化GRU门控: 减少门数量

六、跨技术栈架构
================================================================================

架构: 虚拟设备 → J2EE接口层(8080) → Python算法层(5000)

【Python服务 (Flask)】
- 端口: 5000
- 接口:
  - GET /health - 健康检查
  - POST /api/predict - 单条预测
  - POST /api/batch_predict - 批量预测

【J2EE服务 (Spring Boot 3.2)】
- 端口: 8080
- 接口:
  - POST /api/fetal/heartbeat - 接收设备数据
  - GET /api/fetal/status - 系统状态
  - GET /api/fetal/stats - 检测统计
  - GET /api/fetal/history - 检测历史

【虚拟设备】
- 支持三种信号类型: normal / suspicious / abnormal
- 符合ISO 11073-10407标准
- 实时生成FHR+UC数据并发送至J2EE服务器

九、技术亮点
================================================================================

1. 防数据泄露划分 - 按胎儿(record_id)分组划分，确保同一胎儿样本不会出现在不同数据集中

2. 科学预处理流程 - 中值滤波去除尖峰伪影 + 三次样条插值补全缺失值

3. 混合特征融合 - 深度可分离卷积自动特征 + 5维临床手工特征

4. 轻量化设计 - 深度可分离卷积 + SE注意力 + 通道注意力 + L1正则化

5. 跨技术栈协作 - J2EE-Python混合架构，兼顾企业级稳定性和算法灵活性

6. 完整演示系统 - 虚拟设备 + RESTful接口 + 前端可视化
